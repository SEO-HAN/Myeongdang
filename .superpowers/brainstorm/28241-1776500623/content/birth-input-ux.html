드럼 피커 (Drum Roller)

iOS 네이티브와 동일한 스크롤 UX. 연도·월·일을 한 화면에서 처리. 별도 키보드 팝업 없음.

장점

  • 키보드 불필요
  • 모바일 네이티브 느낌
  • 실수 선택 방지

단점

  • 구현 복잡도 높음
  • 웹 커스텀 피커 품질 차이

스마트 그리드 (현재 방식 업그레이드)

연도를 슬라이더로 변경. 해당 월의 실제 날수만 표시. 터치 타겟 최소 44px 보장.

미니 캘린더 + ± 연도 버튼

연도는 +/- 버튼으로 조작. 달력 형태로 일 선택. 가장 범용적이고 직관적인 UI.

장점

  • 가장 친숙한 패턴
  • 요일 확인 가능
  • 날짜 실수 없음

단점

  • ±1씩 연도 이동 느림
  • 달력 렌더링 복잡도

현재 UX 핵심 문제점:
① 연도 직접 입력 시 숫자 키보드 팝업 → 폼이 가려짐
② 일 그리드 31개 고정 → 2월 29일, 30일, 31일 선택 가능 (잘못된 날짜)
③ 시간 24셀이 너무 작아 터치 실수 발생 가능 (현재 ~36px, 권장 44px)
