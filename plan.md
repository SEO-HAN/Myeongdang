# 📋 명당지도 — 배포 가능 수준 서비스 완성 계획

## 🔍 현황 진단 (As-Is)

**에이전트 환경**
- `.claude/settings.json` hooks 시스템 존재하나 단일 에이전트 설계
- rules/ 파일이 있으나 UX, 성능, 테스트 기준 없음
- 멀티-에이전트 협업 구조 없음

**UX/UI 치명적 공백**
- 온보딩 이탈율 예상 70~80% (생년월일 입력이 첫 진입 장벽)
- 결과 → 지도 연결 동선 약함 (CTA 1개뿐)
- 바이럴 루프 없음 (공유 후 피드백 없음)
- 일진 배너가 사주 입력 유저에게만 노출 → Hook 기회 손실
- 장소 상세에 리뷰/댓글 없음 → 재방문 이유 없음

**사주 엔진 정확도 한계**
- 절기 날짜 하드코딩 (±1일 오차 허용)
- 지장간(地藏干) 오행 미반영 → 계산 정확도 ↓
- 용신(用神)/합충(合沖) 분석 없음
- 한국 대표 사주 서비스 (사주팔자, 운세앱) 대비 정밀도 부족

**서비스 완성도**
- G단계(리뷰, 관리자, 푸시) 미구현
- 레이더 차트 연결 누락
- SNS 공유 카드 미완성

## 🏗️ 전체 계획 — 4개 Phase

---

### **Phase A — 멀티 에이전트 하네스 구축** (기반)

Claude 공식 Agent SDK 패턴 기반으로, 이 프로젝트 전용 **5개 전문 에이전트** 환경을 설계합니다.

```
Orchestrator (CLAUDE.md)
    ├── 🔮 SajuExpert Agent    — 사주 도메인 전문가
    ├── 🎨 UXDesigner Agent    — UX/UI 설계 & 검증
    ├── ⚡ DevEngineer Agent   — Next.js/Supabase 구현
    ├── 🧪 QAVerifier Agent    — 계산 정확도 & 품질 게이트
    └── 📊 PMAnalyst Agent     — 지표 분석 & 우선순위
```

**구체적 작업:**
- CLAUDE.md 전면 재설계: 에이전트별 역할·권한·금지사항 정의
- `.claude/rules/` 4개 파일 신규 추가: `ux-design.md`, `saju-domain-deep.md`, `performance.md`, `testing-strategy.md`
- Hooks 강화: Stop 게이트에 사주 정확도 자동 검증 + UX 일관성 체크 추가
- **Skill 파일 3개 신규 생성** (`skill-creator` 활용):
  - `saju-accuracy-checker`: 만세력 기준 계산 결과 검증 skill
  - `ux-flow-reviewer`: 사용자 여정 & 이탈 포인트 분석 skill
  - `deployment-readiness`: 배포 전 체크리스트 자동 실행 skill

### **Phase B — UX/UI 혁신 설계 & 구현**

#### B-1. 사용자 여정 전면 재설계

**현재 (문제):**
```
진입 → [지도 보임] → 하단 배너 클릭 → [온보딩: 생년월일 입력] → [결과] → 지도로
```
이탈 포인트: 생년월일 입력이 너무 이름 (인지 부하 高)

**목표 (3단계 Progressive Disclosure):**
```
Stage 1 — 즉각 가치 (0~30초)
  진입 → 오늘의 명당 3곳 즉시 노출 + 오늘 일진 배너 (로그인 불필요)
  → "내 사주에 맞게 필터하기" CTA (부드러운 유도)

Stage 2 — 개인화 진입 (30초~2분)
  오행 퀴즈 3문항 (이미지 선택 방식, 30초 완료 가능)
  → 즉시 오행 분석 결과 + 맞춤 명당 지도 필터 적용
  → "더 정확한 분석" → 생년월일 입력 유도

Stage 3 — 딥 인게이지먼트 (재방문)
  카카오 로그인 → 저장된 사주 복원 → 일진 알림 수신
  → 방문 체크인 → 리뷰 작성 → 친구 공유
```

#### B-2. 핵심 화면 & 인터랙션 개선 (12개 항목)

| # | 화면 | 개선 내용 | 목표 지표 |
|---|---|---|---|
| 1 | 지도 메인 | 오늘 일진을 비로그인에도 노출, 미니 카드 형태 | 체류시간 +40% |
| 2 | 온보딩 | 오행 퀴즈 → 생년월일 순서 전환 | 완료율 +35% |
| 3 | 결과 페이지 | 레이더 차트 연결, 대운 미리보기, 공유 CTA 3개 | 공유율 +60% |
| 4 | 바텀시트 | 리뷰 섹션 추가, 방문 체크인 버튼, 스냅 포인트 2단계 | 재방문율 +25% |
| 5 | 장소 상세 | 별점/리뷰, 방문자 수, "나와 같은 오행 추천" 섹션 | 전환율 +30% |
| 6 | 공유 카드 | Canvas API 완성, 오행 비주얼 카드 (카카오/인스타용) | 바이럴 계수 K>1 |
| 7 | 일진 배너 | 확장 시 오늘의 추천 명당 TOP3 연동 | 클릭율 +50% |
| 8 | 필터바 | "내 주변 명당" 탭 추가 (GPS), 운 유형별 탭 | 필터 사용율 +45% |
| 9 | 랜딩/OG | 유형별 맞춤 OG 이미지 (오행별 비주얼 시스템) | CTR +30% |
| 10 | 푸시 알림 | 대길 일진 당일 오전 8시, 재방문 넛지 | D7 리텐션 +20% |
| 11 | 관리자 대시보드 | 장소 CRUD, 신뢰도 편집, 리뷰 승인 | 운영 효율화 |
| 12 | 오프라인/PWA | 마지막 조회 명당 오프라인 캐시 | 앱 설치 전환 +15% |

### **Phase C — 사주 엔진 정확도 고도화**

**한국 대표 사주 서비스 벤치마킹 기준:**
- 사주팔자(sajupalga.com), 운세앱, 만세력 앱 계산법 분석
- 자평진전(子平眞詮) + 명리정종(命理正宗) 기준 적용

**구체적 개선:**

```
C-1. 절기 정밀도 개선
  현재: 고정 날짜 테이블 (±1일 오차)
  개선: korean-lunar-calendar 패키지로 실제 절기 시각 계산
        → 1950~2100년 정확한 입춘/절기 시각 반영

C-2. 지장간(地藏干) 오행 반영
  현재: 천간(1) + 지지(1) = 기둥당 2글자만 계산
  개선: 지지 내 지장간 포함 → 기둥당 여기장간 비율 반영
        예) 子 → 壬(100%) / 丑 → 癸(30%)己(60%)辛(10%)
        → 오행 분포 계산 정밀도 대폭 향상

C-3. 용신(用神) 기초 분석
  현재: 없음
  개선: 일간 기준 억부법(抑扶法) 적용 기초 버전
        → "이런 기운이 필요한 분입니다" 설명 강화

C-4. 합충(合沖) 분석
  현재: 없음
  개선: 천간합(天干合), 지지충(地支沖) 탐지
        → "갑기합으로 토 기운 변화" 등 서술 풍부화

C-5. 35개 테스트 → 100개 테스트 확장
  역사적 검증: 1900~2050 전체 범위 경계값 테스트
  절기 경계: 입춘 당일 정확 시각 기준 테스트
  지장간: 12지지별 지장간 비율 검증
```

### **Phase D — 배포 가능 수준 완성**

```
D-1. G단계 전체 구현
  - reviews 테이블 + 별점/댓글 UI
  - 관리자 대시보드 (/admin)
  - Web Push API (VAPID 키 발급 + 구독 관리)
  - Supabase Storage 이미지 업로드

D-2. 성능 & SEO
  - Core Web Vitals: LCP <2.5s, CLS <0.1, FID <100ms
  - sitemap.xml 동적 생성 (30개+ 장소 URL)
  - 구조화 데이터 강화 (LocalBusiness, Review schema)
  - 이미지 최적화 (WebP, AVIF)

D-3. 모니터링
  - Vercel Analytics 연동
  - 사주 계산 오류 Sentry 트래킹
  - 일진 API 업타임 모니터링

D-4. 배포 파이프라인
  - GitHub Actions CI/CD
  - 사주 엔진 테스트 자동화 (PR 시 필수 통과)
  - Vercel Preview → Production 배포 플로우
```
